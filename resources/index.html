Programming is learned by writing programs

- Brian Kernighan

Every link and resource on this page is handpicked and curated by a team of very experienced programmers. We have done the hard work of only picking the best resources for you so that you don't have to go hunting for it on the internet. Just concentrate on Learning to Code.

It is very important, especially in the initial stages of learning to program that you spent as minimum time as possible on going to and fro over resources and just stick with the ones recommended here. Also, not all of it is supposed to be consumed at once. Slowly go through these materials one by one at your own pace.

How to ask Questions and get Answers

Google everything. I mean everything. Google your dreams, google your problems. Don't ask a question before you Google it. You'll either find the answer or you'll come up with a better question.

- Austin Kleon, Steal Like An Artist

As you start learning to code, there'll be a million questions and a billion little things will go wrong. A huge majority of your questions can be answered by yourself by looking for solutions on the internet. Google is a programmers' best friend.

Googling whenever you encounter a problem will train you in finding answers much more efficiently in the future. This is the most important skill that you need to get really good at very fast.

But, if you can't find answers by Googling, and if your problem is indeed unique then Stack Overflow is your next best friend. It is a very strict and serious question answer community for programmers.

To participate effectively in Stack Overflow, you need to keep a couple of things in mind before firing off your questions right away. This also applies equally to asking questions in our Learn to Code community. Read this before you do anything else: Writing the perfect question.

Basically this is what programmers do the majority of their time. Debugging problems : )

How to use Git and GitHub

Git and GitHub are becoming indispensable tools for a Software professional. This Udacity course will prepare you with everything that you need to know to use these tools and start collaborating with other developers.

Books

If you are learning Python, one of the best introductory books on the planet on programming is How to think like a Computer Scientist. It starts with the very basics and upon completion, you'll have a solid grasp of the fundamentals of programming.

If you are learning Ruby, then Chris Pine's Learn to program is a great book.

And if you are learning Javascript then Eloquent Javascript is a very highly recommeded book.

How to use Linux and the Commandline

Increasingly, the basics of how to use Linux and the commandline is becoming essential for a working programmer. The Linux Command Line by William Shotts and published by no starch press will give you a thorough and practical introduction on how to use Linux from the commandline. You only need to go through Part 1 of the book (the first 121 pages) to understand the basics.

Algorithms and Datastructures

Going through HackerRank will familiarise you with the fundamentals of Computer science. But if you are not prepared for it, it can be daunting and intimidating. Prepare for it using this excellent book on Algorithms and Datastructures.

Online Communities

As you slowly take shape as a programmer, there are developer oriented communities out there that can help you grow and improve yourself. Just reading the discussions on Hacker News in itself will affect the way you start thinking and approaching problems, and if you can participate in the discussions, the better. /r/programming and /r/learnprogramming tends to be on the lighter side, but have really good communities.

Offline Communities

It is a good habit to participate in offline developer meetups. You get to meet a lot of really good programmers and have discussions and chats with them. For example in Bangalore, there is Bangpypers for Python, Bangalore Ruby User's group for Ruby, BLRDROID for Android etc ... Search for your interest in meetup.com and try attending one of the meetups related to your interest.
